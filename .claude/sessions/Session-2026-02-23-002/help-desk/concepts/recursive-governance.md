# Recursive Governance

The QMS governs its own evolution. When a process fails:

1. An **Investigation** analyzes the failure
2. **CAPAs** identify process improvements
3. Child **CRs** implement those improvements (often as SOP updates)
4. The updated SOPs govern future work

The system that discovers a failure and the system that fixes it are the same system.
