# Qualified Commits

A qualified commit is a specific git hash with three properties:

1. **CI-verified** -- all tests pass at this exact commit
2. **RS-backed** -- an EFFECTIVE Requirements Specification exists
3. **RTM-verified** -- an EFFECTIVE RTM maps all requirements to evidence

The hash is recorded in the RTM's Qualified Baseline section. "TBD" means qualification has not occurred.

See `QMS-Docs/09-Code-Governance.md`.
